people.py

Creates the Bonobo pipeline for transforming people data from CSV to JSON-LD.

  1. services :

    1. MakeLinkedArtPerson

    2. people_groups

    3. trace_counter

    4. fs.data.people

    5. currencies : a dict like { 'Â£' : 'gb pounds', 'livres' : 'fr livres' }

    6. materials_map : a dict of frozensets like the following

      ('Painting', frozenset(\{''})) : {
    'classified_as (object type) (primary)': '300033618',
    'classified_as (object type) (secondary)': '', 'editorial
notes': '',
    'made_of (primary)': '',
    'made_of (support)': '',
    'materials': '',
    'n': '548703',
    'object_type': 'Painting',
    'technique': '300054216'
}

      Data come from the data/common/materials_map.csv

    7. Unique_locations : a dict of dicts with places and their canonical names. Data come from data/common/unique_locations.json

    8. vocab_instance_map and vocab_type_map : see the files under data/common with the same name.

  2. graph :

    1. MatchingFiles : get all files matching a specific pattern

    2. CurriedCSVReader : line by line reader of the csv files.

    3. KeyManagement : for each column of the csv files provided, run process the values. (remove,rename, group values etc)

      1. currently this step is skipped (pass-ed)

    4. ExtractKeyedValue : make a copy of the original dict

    5. AddPerson : all sorts of normalization/transformation functionality that a single record may need

    6. add_person_or_group_chain : adds serialization nodes

    7. add_places_chain : adds serialization nodes

  3. Questions :

    1. What are static instances and why are they used?

      1. All the cromulent objects accessed by the workflow.

    2. Why are static instances serialized after the people graph has concluded?

      1. In order to serialize only those accessed.
